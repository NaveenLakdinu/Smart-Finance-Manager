# Transfer Feature Implementation Plan

## Overview

Add an account-to-account transfer function to the Multi-Account Dashboard. Users select a source and destination account, enter an amount, and the app atomically debits the source and credits the destination, then logs the transfer. Includes a transfer history view showing past transfers.

---

## Current State (from exploration)

- **`MultiAccountDashboardActivity.java`** (392 lines) — `cardTransfer` click listener is a stub: `Toast "Transfer - Coming Soon"`
- **`AccountInfo`** (private static inner class) — fields: `name`, `balance`, `number` — no document ID stored
- **Firestore structure**: `users/{uid}/accounts/{auto-doc-id}` with fields `name`, `balance`, `accountNumber`
- **No transfer or transaction history** exists anywhere in the app
- **No Firestore `runTransaction()`** usage — app uses `WriteBatch` for atomicity in 2 places (`RegisterActivity`, `RoleUpgradeActivity`)
- **No centralized helper** — all Firestore access is inline `FirebaseFirestore.getInstance()`

### Step 4: Create `TransferActivity.java`

Key responsibilities:

1. **Parse Intent extras** — extract account arrays, build `AccountInfo` list
2. **Source/Dest account picker** — two card views that open an AlertDialog to select account (same pattern as `showAccountSwitchDialog()`)
3. **Amount validation** — must be > 0, must not exceed source balance
4. **Transfer execution** — use Firestore `runTransaction()` for atomicity:

```java
FirebaseFirestore.getInstance().runTransaction(transaction -> {
    DocumentSnapshot sourceDoc = transaction.get(sourceRef);
    DocumentSnapshot destDoc = transaction.get(destRef);
    
    double sourceBalance = sourceDoc.getDouble("balance");
    double destBalance = destDoc.getDouble("balance");
    
    if (sourceBalance < amount) {
        throw new FirebaseFirestoreException("Insufficient funds",
            FirebaseFirestoreException.Code.FAILED_PRECONDITION);
    }
    
    transaction.update(sourceRef, "balance", sourceBalance - amount);
    transaction.update(destRef, "balance", destBalance + amount);
    
    return null;
}).addOnSuccessListener(aVoid -> {
    // Log transfer to history subcollection
    logTransfer(sourceName, destName, amount, note);
    // Show success, finish activity
}).addOnFailureListener(e -> {
    // Show error toast
});
```

5. **Transfer history logging** — write to `users/{uid}/transfers` subcollection:

```java
private void logTransfer(String from, String to, double amount, String note) {
    FirebaseUser user = FirebaseAuth.getInstance().getCurrentUser();
    if (user == null) return;
    
    Map<String, Object> transfer = new HashMap<>();
    transfer.put("fromAccount", from);
    transfer.put("toAccount", to);
    transfer.put("amount", amount);
    transfer.put("note", note != null ? note : "");
    transfer.put("timestamp", FieldValue.serverTimestamp());
    
    FirebaseFirestore.getInstance()
        .collection("users").document(user.getUid())
        .collection("transfers")
        .add(transfer);
}
```

6. **On return to dashboard** — `onResume()` already calls `loadAccountsFromFirestore()` (existing pattern), so balances refresh automatically.

### Step 10: Create `TransferHistoryActivity.java`

Key responsibilities:

1. **Load transfers** from `users/{uid}/transfers` ordered by `timestamp` descending:

```java
private void loadTransferHistory() {
    FirebaseUser user = FirebaseAuth.getInstance().getCurrentUser();
    if (user == null) return;
    
    FirebaseFirestore.getInstance()
        .collection("users").document(user.getUid())
        .collection("transfers")
        .orderBy("timestamp", Query.Direction.DESCENDING)
        .get()
        .addOnSuccessListener(queryDocumentSnapshots -> {
            List<TransferInfo> transfers = new ArrayList<>();
            for (QueryDocumentSnapshot doc : queryDocumentSnapshots) {
                String from = doc.getString("fromAccount");
                String to = doc.getString("toAccount");
                Double amount = doc.getDouble("amount");
                String note = doc.getString("note");
                Timestamp ts = doc.getTimestamp("timestamp");
                if (from != null && to != null && amount != null) {
                    transfers.add(new TransferInfo(from, to, amount, note, ts));
                }
            }
            if (transfers.isEmpty()) {
                txtEmptyState.setVisibility(View.VISIBLE);
                recyclerTransfers.setVisibility(View.GONE);
            } else {
                txtEmptyState.setVisibility(View.GONE);
                recyclerTransfers.setVisibility(View.VISIBLE);
                recyclerTransfers.setAdapter(new TransferAdapter(transfers));
            }
        });
}
```

2. **Inner model class** `TransferInfo` with fields: `fromAccount`, `toAccount`, `amount`, `note`, `timestamp`
3. **Back button** — `finish()` on toolbar back arrow click
4. **Empty state** — show "No transfers yet" when list is empty

No schema migration needed. One new optional subcollection:

```
users/{uid}/transfers/{auto-doc-id}
  ├─ fromAccount: String
  ├─ toAccount: String
  ├─ amount: double
  ├─ note: String
  └─ timestamp: Timestamp (server)
```

This is additive — existing data is untouched.

## Edge Cases & Validation

| Case | Handling |
|------|----------|
| User has < 2 accounts | Disable transfer button, show "Add at least 2 accounts" toast |
| Source == Destination | Disable transfer button, show "Cannot transfer to same account" |
| Amount > source balance | Block submit, show "Insufficient funds" |
| Amount <= 0 | Block submit, show "Enter a valid amount" |
| Empty amount field | Block submit |
| Firestore transaction fails | Show error toast, balances unchanged |
| Network error | Show toast with error message |
