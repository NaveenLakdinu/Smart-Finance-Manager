# Fix Plan: Expense Claims Crash + Payslip Generation

## Issue 1: Expense Claims Crash

### Root Cause Analysis

The crash occurs when clicking buttons in `ExpenseClaimsActivity` which opens `ExpenseClaimListActivity`. Two issues found:

1. **`ExpenseClaimsActivity.java` line 118**: Uses `android.app.AlertDialog.Builder` (system dialog) with `R.style.Theme_SmartFinance_Dialog`. The theme extends `ThemeOverlay.MaterialComponents.Dialog.Alert` which requires `AppCompatActivity` theme context. Using `android.app.AlertDialog.Builder` instead of `androidx.appcompat.app.AlertDialog.Builder` can cause `IllegalStateException` when the style isn't compatible.

2. **`ExpenseClaimListActivity.java`**: The `activity_expense_claims.xml` layout uses `CoordinatorLayout` as root with a `ScrollView` containing a `RecyclerView` (`nestedScrollingEnabled="false"`). When the RecyclerView adapter is set and Firestore loads data, the RecyclerView inside ScrollView can trigger measure/layout issues. Also, `setupBackButton()` (line 87) calls `findViewById(R.id.btnBack).setOnClickListener(...)` without null check.

### Fix Steps

**File: `ExpenseClaimsActivity.java`**
- Line 118: Change `android.app.AlertDialog.Builder` → `androidx.appcompat.app.AlertDialog.Builder`
- Add null checks for `txtTotalPendingClaims` before `setText()`

**File: `ExpenseClaimListActivity.java`**
- Line 87: Add null check: `View btnBack = findViewById(R.id.btnBack); if (btnBack != null) btnBack.setOnClickListener(...)`
- Remove duplicate `setupBackButton()` call (already handled in `initViews()`)
- Wrap `loadClaimsFromFirestore()` in try-catch for safety
- Add `addOnFailureListener` to the Firestore query

## Issue 2: Add Payslip Generation + Salary Calculation

### Current State
- `WorkerPayslipActivity.java` only READS payslips from Firestore
- No way to CREATE/GENERATE payslips
- No auto-calculation from `worker_profile.monthlySalary`

### What Needs to Be Added

1. **"Generate Payslip" button** in `activity_worker_payslip.xml` (next to Download button)
2. **Generate Payslip dialog** with fields:
   - Month/Year selector (spinner)
   - Days worked (number input)
   - Overtime hours (number input)
   - Transport allowance (auto-calculated or manual)
   - Performance bonus (manual input)
   - All other values auto-calculated from `monthlySalary`
3. **Salary calculation logic**:
   - `basicSalary` = `worker_profile.monthlySalary`
   - `epfDeduction` = `basicSalary * 0.08` (8% EPF in Sri Lanka)
   - `incomeTax` = calculated from Sri Lankan tax brackets
   - `transportAllowance` = default 10% of basic or manual
   - `grossEarnings` = basic + transport + bonus + overtime pay
   - `netSalary` = gross - deductions
4. **YTD calculation**: Sum all payslips in current year
5. **Save to Firestore** `users/{uid}/payslips`

### Sri Lankan Salary Calculation
```
basicSalary = worker_profile.monthlySalary
epfDeduction = basicSalary * 0.08 (employee EPF)
transportAllowance = basicSalary * 0.10 (10% default)
overtimePay = (basicSalary / 200) * overtimeHours * 1.5
grossEarnings = basicSalary + transportAllowance + performanceBonus + overtimePay
incomeTax = based on monthly taxable income brackets
netSalary = grossEarnings - epfDeduction - incomeTax
```
