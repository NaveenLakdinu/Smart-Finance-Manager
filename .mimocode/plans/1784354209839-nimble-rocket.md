# Plan: Complete Subscription Management System with Payment Notifications

## Current State Analysis

**What exists:**
- `SubscriptionManagerActivity` — Hub with 3 buttons: Add Plan (opens wrong activity), My Plans (toast "Coming Soon"), Report (toast "Coming Soon")
- `SubscriptionActivity` — This is a **notification preferences form** (email/SMS/push checkboxes), NOT a subscription add form. It saves to `UserRegistrationData` singleton, not Firestore.
- `Subscription.java` — Model: `name`, `renewDate`, `status`, `logoType`, `createdAt` (missing `amount`, `paymentDay`, `billingCycle`, `documentId`)
- `RecentSubscriptionAdapter` — RecyclerView adapter displaying subscriptions (works, but only used nowhere currently)
- Layout `activity_subscription_manager_function.xml` — Has hardcoded "Netflix Premium" sample card
- Firestore collection: `users/{uid}/subscriptions` (referenced in AGENTS.md)

**What's broken/incomplete:**
1. "Add Plan" button opens `SubscriptionActivity` (notification prefs) instead of a subscription form
2. "My Plans" and "Report" buttons are stubs
3. No Firestore read/write for subscriptions
4. No payment date notification system
5. `Subscription` model is missing key fields (`amount`, `paymentDay`, `billingCycle`)
6. Dashboard hero card shows hardcoded "LKR 4,500.00"

## Implementation Plan

### Step 1: Update `Subscription.java` Model

**File: `Subscription.java`**

Add missing fields:
```java
private String documentId;
private double amount;        // monthly cost in LKR
private int paymentDay;       // day of month (1-31)
private String billingCycle;  // "Monthly" or "Yearly"
```

Update constructor, getters/setters. Add `toMap()` helper for Firestore writes.

### Step 2: Create `AddSubscriptionActivity.java` + Layout

**New file: `AddSubscriptionActivity.java`**

Form fields:
- **Plan Name** — EditText (e.g., "Netflix Premium")
- **Monthly Cost** — EditText, number/decimal input (e.g., "1500")
- **Payment Date** — DatePicker or day-of-month spinner (1-31)
- **Billing Cycle** — RadioGroup: Monthly / Yearly
- **Logo Type** — Spinner or chips: Netflix, Spotify, YouTube, ChatGPT, Other

On save:
1. Validate all fields
2. Create `Subscription` object
3. Write to `users/{uid}/subscriptions` in Firestore
4. Schedule payment notifications (3 alarms: day-2, day-1, day-0)
5. Finish activity

**New layout: `activity_add_subscription.xml`**

Follow the dark glassmorphic theme. Use `MaterialCardView` with input fields styled like existing forms (see `RegisterBillActivity` pattern).

### Step 3: Create `SubscriptionListActivity.java` + Layout

**New file: `SubscriptionListActivity.java`**

- RecyclerView displaying all subscriptions from `users/{uid}/subscriptions`
- Uses existing `RecentSubscriptionAdapter`
- Swipe-to-delete with undo
- Tap to view/edit details
- Empty state: "No subscriptions yet. Tap + to add one."

**New layout: `activity_subscription_list.xml`**

- Header with back button + title "My Subscriptions"
- RecyclerView
- Empty state view (hidden when list has items)

### Step 4: Create `SubscriptionDetailActivity.java` + Layout

**New file: `SubscriptionDetailActivity.java`**

Shows full details of one subscription:
- Plan name, amount, payment day, billing cycle, status
- Edit button → opens edit form (reuse AddSubscription layout)
- Delete button → confirm dialog → delete from Firestore → cancel notifications
- "Mark as Paid" button → updates status, advances payment date to next cycle

### Step 5: Create `SubscriptionReportActivity.java` + Layout

**New file: `SubscriptionReportActivity.java`**

- Total monthly cost (sum of all active subscriptions)
- List grouped by billing cycle
- Bar chart of top subscriptions by cost (using MPAndroidChart, already in project)
- Export as PDF button (using iTextPDF, already in project)

### Step 6: Payment Notification System (2 Days Before → Payment Day)

**New file: `SubscriptionNotificationScheduler.java`**

Uses `AlarmManager` + `BroadcastReceiver` (same pattern as `NotificationReceiver` already in project).

**Scheduling logic:**
When a subscription is saved/updated with `paymentDay`:
1. Calculate next payment date from today
2. Schedule 3 separate alarms:
   - **Day -2**: "Your [Plan Name] payment of LKR [amount] is due in 2 days"
   - **Day -1**: "Your [Plan Name] payment of LKR [amount] is due tomorrow"
   - **Day 0** (payment day): "Your [Plan Name] payment of LKR [amount] is due today!"
3. Each alarm fires at 9:00 AM on that day
4. Store pending alarm IDs in Firestore document for cancellation on delete

**New file: `SubscriptionAlarmReceiver.java`**

Extends `BroadcastReceiver`. On receive:
- Reads subscription details from intent extras
- Shows notification via `NotificationHelper`
- If today is payment day, auto-advance the payment date to next cycle and reschedule

**Cancel logic:**
When a subscription is deleted, cancel all pending alarms using `AlarmManager.cancel()`.

### Step 7: Wire Up `SubscriptionManagerActivity`

**File: `SubscriptionManagerActivity.java`**

Load total monthly cost from Firestore and display in hero card (replace hardcoded "LKR 4,500.00").

Add:
