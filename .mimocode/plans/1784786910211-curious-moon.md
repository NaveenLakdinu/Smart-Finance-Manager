# Plan: Loan Compare — Min 2, Max 10

## Goal
Allow users to compare 2–10 loans (currently defaults to 3, no min/max enforced).

---

## Files to Modify

### 1. `app/src/main/java/com/example/smartfinancialmanagement/LoanCompareActivity.java`

**Constants to add:**
```java
private static final int MIN_LOAN_OPTIONS = 2;
private static final int MAX_LOAN_OPTIONS = 10;
```

**`onCreate()` changes:**
- Change default loop from `i < 3` → `i < MIN_LOAN_OPTIONS` (starts with 2 cards)

**`addLoanOptionCard()` changes:**
- Before adding: check `loanCardsContainer.getChildCount() >= MAX_LOAN_OPTIONS`
  - If at max, show Toast: "Maximum 10 loan options allowed" and return
- After adding: call `updateAddButtonVisibility()`

**New method `updateAddButtonVisibility()`:**
```java
private void updateAddButtonVisibility() {
    int count = loanCardsContainer.getChildCount();
    btnAddOption.setVisibility(count >= MAX_LOAN_OPTIONS ? View.GONE : View.VISIBLE);
}
```

**`addLoanOptionCard()` — remove button logic:**
- Show remove button when `loanCardsContainer.getChildCount() > MIN_LOAN_OPTIONS` (i.e., 3+ cards → can remove back to 2)
- After removal: call `updateAddButtonVisibility()` and `refreshOptionTitles()`

**New method `refreshOptionTitles()`:**
- Iterate all children in `loanCardsContainer`
- Set title text to "Option A", "Option B", etc.
- Show/hide remove buttons based on whether count > MIN_LOAN_OPTIONS

**`validateAllCards()` changes:**
- After checking childCount > 0, also check `childCount >= MIN_LOAN_OPTIONS`
  - If < 2 valid cards, show Toast: "Please add at least 2 loan options to compare"

### 2. `app/src/main/res/layout/activity_loan_compare.xml`
- No structural changes needed (add button visibility controlled from Java)

### 3. `app/src/main/res/layout/item_loan_compare_card.xml`
- No changes needed

## Execution Order
1. Add constants to `LoanCompareActivity.java`
2. Update `onCreate()` default count
3. Add `updateAddButtonVisibility()` and `refreshOptionTitles()` methods
4. Update `addLoanOptionCard()` with max check + remove logic
5. Update `validateAllCards()` with min check
6. Build verification: `./gradlew assembleDebug`

## Verification
1. Build compiles without errors
2. On launch: 2 loan cards appear (not 3)
3. "Add Another Option" visible → click adds card (up to 10)
4. At 10 cards: "Add Another Option" hidden, Toast shown if triggered
5. Remove button visible on cards 3–10, clicking removes and re-indexes titles
6. Cannot remove below 2 cards
7. "Generate Comparison Report" requires at least 2 valid cards
