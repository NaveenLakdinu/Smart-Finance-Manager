# Fix: Student Dashboard Not Showing Current Savings Amount

## Root Cause

`StudentDashboardActivity` has **no savings widget and no Firestore data loading** for the `currentSavings` field.

Two problems:

1. **Layout XML** (`activity_student_dashboard.xml`): The "CURRENT BALANCE" card (line 222-420) displays a hardcoded `"LKR 48,250.00"` in `txtCurrentBalanceValue` (line 265). There is no `cardSavingsWidget`, `txtCurrentSavingsValue`, or `btnUpdateSavings` — the savings widget that other dashboards have is completely absent.

2. **Java Activity** (`StudentDashboardActivity.java`): Has no `setupSavingsWidget()`, `loadSavingsFromFirestore()`, or `showUpdateSavingsDialog()` methods. The only Firestore read is `loadAchievementData()` (line 118) which reads from the `savings` subcollection for achievement badges — it does NOT read the root user document's `currentSavings` field.

Compare: Worker/MultiAccount/Hybrid dashboards all have a `cardSavingsWidget` in their XML and wire it up with `setupSavingsWidget()` → `loadSavingsFromFirestore()` → Firestore `getString("currentSavings")`.

## Plan

### Step 1: Add savings widget to `activity_student_dashboard.xml`

**File:** `app/src/main/res/layout/activity_student_dashboard.xml`

Insert the standard savings widget `MaterialCardView` (id `cardSavingsWidget`) between the CURRENT BALANCE card closing tag (line 420) and the "Quick Actions Title" section (line 422). Use the same XML structure from `activity_worker_dashboard.xml` lines 211-285:

- `MaterialCardView` with id `cardSavingsWidget`, corner radius 20dp, glass card styling
- Horizontal `LinearLayout` containing:
  - Money emoji icon (44dp, `bg_function_icon_green`)
  - Vertical layout: "CURRENT SAVINGS" label + `txtCurrentSavingsValue` (default "LKR 0.00")
  - "UPDATE" pill button (`btnUpdateSavings`)

### Step 2: Add Firestore data loading methods to `StudentDashboardActivity.java`

**File:** `app/src/main/java/com/example/smartfinancialmanagement/StudentDashboardActivity.java`

Add three methods, copied from `WorkerDashboardActivity.java` (lines 163-258):

1. **`setupSavingsWidget()`** — Finds `txtCurrentSavingsValue`, `btnUpdateSavings`, `cardSavingsWidget` by ID, calls `loadSavingsFromFirestore()`, attaches click listeners for the update dialog.

2. **`loadSavingsFromFirestore(TextView txtValue)`** — Reads `users/{uid}` document, gets `getString("currentSavings")`, parses to double, formats as `"LKR %.2f"`, sets the TextView.

3. **`showUpdateSavingsDialog(TextView txtValue)`** — AlertDialog with numeric EditText, writes back via `.update("currentSavings", String.valueOf(amt))`, refreshes the TextView.

### Step 3: Wire up in `initViews()` and add `onResume()`

**File:** `app/src/main/java/com/example/smartfinancialmanagement/StudentDashboardActivity.java`

- Call `setupSavingsWidget()` at the end of `initViews()` (after line 81, before `loadAchievementData()`)
- Add `onResume()` override that calls `loadSavingsFromFirestore()` to refresh data when returning to the screen

### Step 4: Add required import

Add `import java.util.Locale;` to `StudentDashboardActivity.java` (needed by `String.format(Locale.US, ...)`).
