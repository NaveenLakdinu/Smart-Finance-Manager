## Part 1: Landing Page Redesign (`activity_main.xml` + `MainActivity.java`)

### Current Landing Page
- Full-screen background image (`hero_finance_banner.png`)
- Dark overlay (`dim_overlay`)
- Logo + "RED ANTS" brand label
- "SMART FINANCIAL MANAGEMENT" title
- Bottom panel with Login + Sign Up buttons

### New Landing Page Design
Keep the same structure but add **animated entrance effects**:

#### Animations to Add
1. **Logo fade-in + scale-up** — Logo starts at 80% scale, fades in over 500ms
2. **Brand label slide-up** — Slides up from 20dp below, fades in, delayed 200ms
3. **Title typewriter effect** — Characters appear one by one or fade in word-by-word
4. **Divider line expand** — Width animates from 0dp to 40dp, delayed 400ms
5. **Bottom panel slide-up** — Slides up from bottom over 600ms, delayed 300ms
6. **Login button pulse** — Subtle scale pulse (1.0 → 1.03 → 1.0) on repeat after 1s delay
7. **Sign Up button fade-in** — Fades in after login button appears, delayed 500ms

#### Files to Create
- `res/anim/fade_in.xml` — Generic fade-in (0% → 100% alpha, 500ms)
- `res/anim/slide_up_from_bottom.xml` — Translate Y 100% → 0%, 600ms, decelerate
- `res/anim/slide_up.xml` — Translate Y 20dp → 0dp, 400ms
- `res/anim/scale_fade_in.xml` — Scale 80% → 100% + fade in, 500ms
- `res/anim/expand_width.xml` — Width 0 → 40dp, 400ms
- `res/anim/pulse_repeat.xml` — Scale 1.0 → 1.03 → 1.0, 1000ms, infinite

#### Files to Modify
- `activity_main.xml` — Add `android:visibility="invisible"` to animated elements (start hidden)
- `MainActivity.java` — Load animations in `onCreate()`, apply to views with staggered delays

## Part 3: Dashboard Card Animations

### Card Entrance Animation
When a dashboard loads, cards should animate in with a **staggered fade-in + slide-up** effect.

#### Files to Create
- `res/anim/card_enter.xml` — Translate Y 40dp → 0dp + fade in, 400ms, overshoot interpolator

#### Implementation
In each dashboard's `onCreate()` or `onResume()`, after data loads:
```java
// Example for StudentDashboardActivity
MaterialCardView card = findViewById(R.id.cardSavingsWidget);
card.setAlpha(0f);
card.setTranslationY(40f);
card.animate()
    .alpha(1f)
    .translationY(0f)
    .setDuration(400)
    .setStartDelay(100) // staggered per card
    .setInterpolator(new OvershootInterpolator(1.2f))
    .start();
```

#### Files to Modify (6 dashboards)
- `StudentDashboardActivity.java` — Animate top cards on load
- `WorkerDashboardActivity.java` — Same
- `BusinessDashboardActivity.java` — Same
- `MultiAccountDashboardActivity.java` — Same
- `DashboardActivity.java` — Same
- `StudentWorkerHybridDashboardActivity.java` — Same

## Part 5: Button Micro-Interactions

### Ripple Effect
All `MaterialButton` elements already have Material ripple by default. Ensure no buttons override with `android:stateListAnimator="@null"` that kills the ripple.

### Touch Feedback
For `MaterialCardView` cards used as buttons, ensure `android:clickable="true"` and `android:foreground="?attr/selectableItemBackground"` are set for touch feedback.

#### Files to Review (6 dashboard layouts)
