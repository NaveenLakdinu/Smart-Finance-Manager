### Root Cause

## Issue 3: PDF Reports ÔÇö Add Pie Charts to All Reports

### Scope ÔÇö 4 PDF-generating activities
| Activity | PDF library | Current charts | Action |
|---|---|---|---|
| `LoanReportActivity` | `android.graphics.pdf.PdfDocument` | None (text-only table) | Add PieChart |
| `LoanCompareActivity` | `android.graphics.pdf.PdfDocument` | None (text-only table) | Add PieChart |
| `UtilityReportActivity` | iTextPDF | BarChart already embedded | Add PieChart alongside |
| `AnalyticsActivity` | iTextPDF | BarChart already embedded | Add PieChart alongside |

### Approach
Use **MPAndroidChart** (already in `build.gradle:83`) to create `PieChart` views programmatically, render them to Bitmap, then embed in the PDF.

**For `android.graphics.pdf.PdfDocument` (LoanReport, LoanCompare):**
- Create a `PieChart` view programmatically
- Call `measure()` + `layout()` to size it
- Render to Bitmap via `chart.draw(new Canvas(bitmap))`
- Draw Bitmap onto the PDF Canvas via `canvas.drawBitmap()`
- Start a new PDF page if needed to fit the chart

**For iTextPDF (UtilityReport, Analytics):**
- Same programmatic PieChart creation and Bitmap rendering
- Convert Bitmap to iTextPDF `Image` and `document.add(chartImage)`
- Pattern already used by UtilityReportActivity for its BarChart (lines 295-305)

### Pie Chart Data Per Report
| Report | Pie Chart shows |
|---|---|
| **LoanReportActivity** | Distribution of each loan's principal amount as a proportion of total principal |
| **LoanCompareActivity** | Distribution of each option's total payable amount |
| **UtilityReportActivity** | Distribution of utility bills by amount (each bill type's share of total) |
| **AnalyticsActivity** | Revenue vs Expenses vs Net Profit proportion |
