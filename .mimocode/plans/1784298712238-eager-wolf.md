# UI Modernization Plan — Smart Finance Manager

## Current State

The app has **46+ XML layout files** with **5 conflicting visual themes** across ~85 total layouts. The dominant and most polished theme is the **Dark Navy Glassmorphic** palette (#0A1628 bg, #1A2F50 glass cards, #00D4AA teal accent, #F0F6FF text). The goal is to unify everything under this premium look.

### Theme Map

| Theme | Status | Screens |
|-------|--------|---------|
| Dark Navy Glassmorphic (target) | ~60 screens | Dashboards, Login, PIN, Utility, Invoice, Business, Transfer |
| Light Blue cards | Needs conversion | Student Budget, Saving, Loans, Subscription, Event (~5 screens) |
| Savings Passport parchment | Keep (special) | 1 screen — unique passport metaphor |
| Goal deep dark (#090D1A) | Merge into main | 3 screens — nearly identical to main palette |
| White cards on dark bg | Needs conversion | Account Settings, Financial Reports (~2 screens) |

### Top Issues (by impact)

1. **5 conflicting themes** — jarring transitions between screens
2. **~400+ hardcoded hex colors** across ~35 layout files (no token usage)
3. **Emoji icons** — render inconsistently across Android devices
4. **5 different back button implementations**
5. **5 different button colors** (#00D4AA, #4CAF50, #4ADE80, #1E60FF, #1E3A6E)
6. **20+ typography sizes** with no scale system
7. **White BottomNavigationView** on hybrid dashboard clashes with dark theme
8. **Spinner text invisible** on dark backgrounds (#111827 text on dark bg)
9. **Registration progress bar nearly invisible** (dark on dark)
10. **Landing page "RED ANTS" text is black on dark overlay** (unreadable)

## PHASE 3: Component Consistency

**Risk:** LOW-MEDIUM | **Impact:** HIGH — App feels cohesive

### 3A. Bottom Navigation — Unified Dark Style
- All activities with `BottomNavigationView` → dark bg (#0D1F3C), teal active (#00D4AA), muted inactive

### 3B. Back Button — Single Implementation
- Standardize: 40dp circle with `@drawable/bg_back_circle`, `@drawable/ic_arrow_back` (white tint), `contentDescription`
- Apply across all ~30 screens with back navigation

### 3C. Button Styles — 3 Semantic Styles
- **Primary CTA:** `@color/btn_primary` (#00D4AA) dark text → `@style/Widget.App.Button.Teal`
- **Secondary:** glass card with border → `@style/Widget.App.Button.Outline.Dark`
- **Danger:** red tint/text → dedicated danger style

### 3D. Card Component — Unified MaterialCardView
- All cards → `MaterialCardView`, glass card bg, 1dp border stroke, 16dp corners, 0dp elevation
- Replace `CardView` usages with `MaterialCardView`

### 3E. Dialog Styling
- Add 20dp corner radius, glass card background, scrim
- Apply `@style/Theme.SmartFinance.Dialog` programmatically

### 3F. Currency Symbol Unification
- Replace `$` with `LKR` in goal screens
- Create `@string/currency_symbol` resource for consistency

## PHASE 4: Typography & Spacing Refinement

**Risk:** LOW | **Impact:** MEDIUM-HIGH

### 4A. Typography Scale Application
Apply `textAppearance` styles across all 85 layouts:
- Display: 28sp (hero numbers)
- Headline: 24sp (welcome titles)
- Title Large: 20sp (screen titles)
- Title: 16sp (section headers, card titles)
- Body: 14sp (form labels, input text)
- Caption: 12sp (secondary labels, dates)
- Small: 11sp (micro labels, pills)

### 4B. Hybrid Dashboard Typography Fix
- `activity_student_worker_hybrid_dashboard.xml` — increase all text sizes by 1-3sp to match other dashboards

### 4C. Spacing Standardization
- Apply 4dp grid system: 4/8/12/16/20/24/32/48dp
- Standardize screen horizontal padding to 20dp
- Standardize card padding to 16dp (content) or 24dp (hero)
- Standardize card margins to 16dp

## Risk Mitigation

1. **Build after every phase** — `./gradlew assembleDebug` must pass
2. **Create `ui-modernization` git branch** before starting
3. **One screen at a time** — verify each individually
4. **Test on device** — emulators may not render correctly
5. **Each phase is independently valuable** — can merge incrementally
